# AI Agent Rules & Guidelines (tools)

이 문서는 tools 프로젝트를 개발하는 AI Agent가 준수해야 할 엄격한 수칙을 담고 있습니다.

## 🧪 개발 전략 (Development Strategy)

### 1. 호환성 및 독립성 보장
- **독립 실행**: `tools` 내의 각 도구(`md2html`, `check_version.bat` 등)는 다른 프로젝트에 의존하지 않고 독립적으로 실행되어야 합니다.
- **크로스 플랫폼**: 가능한 한 Windows와 Linux 모두에서 동작할 수 있도록 설계합니다. (배치 파일과 쉘 스크립트 병행 제공 등)

## 🛠️ 코딩 규칙 (Coding Rules)

- **문서 확인**: 수정 전 관련 문서(`README.md` 등)를 읽고 기능의 의도를 파악합니다.
- **코드 무결성**: 실제 동작하는 핵심 로직을 함부로 변경하거나 삭제하지 않습니다.
- **언어 준수**: 모든 사고 과정(Thinking)과 대화는 **한국어**로 진행합니다.

## 📦 커밋 및 문서화 규칙 (Commit & Documentation)

- **커밋 메시지**: 한글로 작성하며, `<type>: <설명>` 형식을 준수합니다. 최대한 상세하게 작성합니다.
   - 예: `feat: check_version.bat에 태그 표시 기능 추가`
   - 예: `fix: html2pdf 변환 시 크롬 프로세스 잔존 문제 수정`
- **커밋 단위**: 논리적으로 완결된 기능 단위로 커밋합니다. (잦은 스냅샷성 커밋 지양)
- **필수 갱신 문서**: 모든 유의미한 변경 시 관련 도구의 사용법이 명시된 문서(`README.md` 또는 상위 프로젝트의 가이드)를 최신화합니다.
