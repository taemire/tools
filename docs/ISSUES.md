### [High] 문서 생성기(md2pdf_v2) 목차 페이지 계산 오류
- **관련 도구**: `md2pdf_v2.bat`, `pdf_analyzer`
- **등록일**: 2025-12-25
- **해결일**: 2025-12-25
- **설명**: `md2pdf_v2.bat`의 `SKIP_PAGES`가 3으로 하드코딩되어 있어, 목차가 2페이지가 아닌 경우(1페이지 또는 3페이지 이상) 본문 섹션 페이지 번호가 어긋나거나 누락됨.
- **영향**:
  - 목차가 1페이지일 경우: 본문 1페이지(전체 3p)의 섹션을 인식 못하고 건너뜀.
  - 목차가 3페이지 이상일 경우: 목차 내용을 본문 섹션으로 오인할 위험 있음.
- **해결 방안**:
  - `pdf_analyzer`에 `detectTocEndPage()` 함수 추가하여 첫 번째 섹션이 나타나는 페이지를 자동 감지
  - `-skip 0` (기본값) 또는 `-skip -1` 시 자동 감지 모드 활성화
  - 수동 지정도 여전히 가능 (양수 값 사용 시)
  - 감지 실패 시 기본값 3 사용 (표지 1p + 목차 2p)
- **상태**: ✅ Resolved
